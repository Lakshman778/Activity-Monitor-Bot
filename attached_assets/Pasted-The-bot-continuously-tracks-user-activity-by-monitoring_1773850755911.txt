The bot continuously tracks user activity by monitoring when members last send a message in the server. If a user has not been active for a configurable number of days (for example, 2 days), the bot will send a warning message in a designated channel. This message notifies the user that they are inactive and asks them to either become active again or submit a leave request explaining their absence.

Users are given the ability to submit inactivity (leave) requests by providing a reason and duration. These requests are reviewed by staff members, who can approve or reject them. Approved requests temporarily exempt the user from inactivity tracking, while rejected requests result in the user continuing to be monitored normally. Staff can view all inactive users and requests in a dedicated staff-only channel, ensuring full transparency and control.

If a user remains inactive beyond a second, longer threshold (also configurable by the server), the bot sends another alert to a staff-only channel. This alert helps moderators decide whether further action, such as kicking the user, is necessary. The bot itself never performs automatic kicks, ensuring that all moderation decisions remain in the hands of human staff.

The system is designed to be efficient and non-intrusive. It avoids spamming users with repeated warnings by only sending alerts at specific stages. If a user becomes active again, their inactivity status is automatically reset. Similarly, if a user’s approved leave period ends, the bot resumes tracking their activity without requiring manual intervention.

To prevent abuse of inactivity status, the bot includes a strict giveaway restriction system. Since the giveaway functionality is handled by a separate bot, direct integration is not possible. Instead, this bot uses Discord’s role and permission system to control access. When a user is marked as inactive or is on approved leave, the bot assigns them a specific role (such as “Inactive” or “On Leave”) and automatically removes their permission to view giveaway channels. This effectively hides the giveaway channel from them, preventing them from participating. Discord supports this type of control through channel-level permissions, where roles can be allowed or denied access to specific channels .

This approach ensures that users cannot exploit inactivity status to avoid participation requirements while still benefiting from server events. If they are inactive, they simply cannot see or access giveaway channels, making the system fair and enforceable without modifying the giveaway bot itself. Additionally, some giveaway systems also support role-based participation requirements, further reinforcing this approach if needed .

All aspects of the bot—including inactivity duration, warning thresholds, channels, and role behavior—are fully configurable for each server. This flexibility allows the bot to adapt to different community rules and moderation styles, making it suitable for a wide range of Discord servers.
